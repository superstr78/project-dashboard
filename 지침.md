## 문서 관리 규칙

### 요구사항 문서 (요구사항.md)
- 사용자가 전달하는 모든 요구사항을 이 문서에 기록
- 변경 이력 섹션에 수정 내용 기록

### 지침 문서 (지침.md)
- 개발 시 따라야 할 규칙과 가이드라인 기록

## 페이지 구조

### 메뉴 구성 (7개)
1. **대시보드** - 전체 현황 요약 (메인)
2. **패키징 현황** - 프로젝트별 패키징 상태
3. **이슈 현황** - 제품별 이슈 (Teams)
4. **빌드 상태** - 빌드 성공/실패
5. **SVN 커밋** - 커밋 비율 통계
6. **Jira** - 이슈 목록
7. **설정** - 연동 설정

### 레이아웃
- 사이드바 메뉴 (좌측 고정)
- 메인 콘텐츠 영역 (우측)
- 반응형 디자인

## 연동 시스템

### Microsoft Teams
- **용도**: 패키징 진행 상황, 제품별 이슈 채널 정보
- **API**: Microsoft Graph API
- **필요 권한**: Channel.Read, ChannelMessage.Read

### Jira
- **용도**: 이슈 관리, 최근 생성/완료 이슈 조회
- **API**: Jira REST API v3
- **필요 정보**: Jira 도메인, API 토큰, 프로젝트 키

### Confluence
- **용도**: 문서 연동
- **API**: Confluence REST API

### SVN
- **용도**: 커밋 비율 통계
- **방식**: svn log 명령어 또는 SVN API

### 빌드 서버
- **용도**: 빌드 성공/실패 상태
- **방식**: (확인 필요 - Jenkins, Azure DevOps 등)

## 기술 스택

### 프론트엔드
- **프레임워크**: React 18 + TypeScript
- **빌드 도구**: Vite
- **라우팅**: React Router v6
- **UI 라이브러리**: Tailwind CSS
- **차트**: Chart.js 또는 Recharts
- **아이콘**: Lucide React

### 백엔드 (선택사항)
- 초기에는 프론트엔드만으로 구현
- API 연동은 클라이언트에서 직접 호출
- 필요 시 Node.js/Express 백엔드 추가

### 데이터 저장
- 설정 정보: LocalStorage
- API 응답 캐싱: React Query

## 개발 가이드라인

### 코드 스타일
- TypeScript strict 모드 사용
- 함수형 컴포넌트 + Hooks 사용
- 컴포넌트 파일명: PascalCase (예: Dashboard.tsx)
- 유틸리티 파일명: camelCase (예: formatDate.ts)

### 명명 규칙
- 컴포넌트: PascalCase
- 함수/변수: camelCase
- 상수: UPPER_SNAKE_CASE
- 타입/인터페이스: PascalCase (I 접두사 없음)

### 커밋 메시지
- feat: 새로운 기능
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 포맷팅
- refactor: 코드 리팩토링
